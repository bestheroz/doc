## 그리디(Greedy) 알고리즘

**핵심 개념**: 매 순간 지역적 최적(Local Optimum)을 선택하여 전체 최적해를 구하는 방식

---

**적용 가능 조건**
1. **탐욕 선택 속성** — 각 단계의 최선 선택이 전체 최적해로 이어짐
2. **최적 부분 구조** — 부분 문제의 최적해가 전체 최적해를 구성

---

**알고리즘 비교**

| 구분 | 그리디 | DP | 완전탐색 |
|------|--------|-----|---------|
| 접근 | 매 순간 최선 | 모든 경우 고려 후 선택 | 모든 경우의 수 탐색 |
| 속도 | O(N) ~ O(N log N) | 느림 | 매우 느림 |
| 최적해(Optimal Solution) | 보장 ❌ | 보장 ✅ | 보장 ✅ |

---

**대표 문제 유형**

| 문제 | 그리디 전략 |
|------|------------|
| 거스름돈 | 큰 단위부터 선택 |
| 회의실 배정 | 종료 시간 빠른 순 |
| 분할 가능 배낭 | 가치/무게 비율 높은 순 |
| 최소 신장 트리 | 가장 작은 간선 선택 |
| 허프만 코딩 | 빈도 낮은 것부터 합치기 |

---

**주의사항**

그리디가 항상 최적해를 보장하지 않으므로 증명이 필요함

```java
// 반례: 동전 {1, 3, 4}로 6원 거슬러주기
// 그리디: 4 + 1 + 1 = 3개
// 최적해: 3 + 3 = 2개
```
